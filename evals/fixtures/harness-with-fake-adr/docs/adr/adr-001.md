# Architecture Decision Records

## ADR-001: Use Repository Pattern for Data Access

**Context**: We need to support multiple data sources in the future.

**Decision**: Use the Repository pattern to abstract data access.

**Consequences**: Additional abstraction layer, but better testability.

**Status**: Accepted

---

## ADR-002: Use Zustand for Global State

**Context**: Local state is insufficient for cross-component sharing.

**Decision**: Adopt Zustand for global state management.

**Consequences**: New dependency, simpler state sharing.

**Status**: Accepted
